01

Genesis · Test Drive 화면기획Genesis · Test Drive spec

Concentrix Catalyst · 서비스 기획Service Design 유럽 롤아웃 · 이탈리아 마켓EU rollout · Italy market 2025 화면기획서(스토리보드)Storyboard

시승 신청을 유저 플로우로 먼저 잡았어요. 어디서 들어오고(진입), 무엇을 고르고, 성공·실패가 어디로 가고, 뒤에서 어떤 시스템과 붙는지까지 한 장에 정리했죠. 그다음 신청 폼을 화면 단위로 펼쳐, 모델 선택·일정·딜러 찾기(Agency Locator)를 번호로 짚고 오른쪽 표에 동작을 1:1로 적었습니다. 흐름과 화면을 같이 봐야 개발이 추측 없이 구현하거든요.I started the test-drive request as a user flow: where people enter, what they pick, where success and failure go, and which backend systems it connects to, all on one page. Then I unfolded the request form screen by screen, pinning model selection, scheduling and the agency locator by number with behavior mirrored 1:1 in the table. Flow and screen have to be read together for engineers to build without guessing.

✎ 실제 산출물에서 발췌 · 유저 플로우의 백엔드 시스템명은 Confidential로 가렸어요. 폼 화면은 왼쪽 번호와 오른쪽 명세 표가 1:1로 대응합니다.Excerpted from the real deliverable · backend system names in the flow are masked as Confidential. On the form screen, the left-side numbers map 1:1 to the spec table on the right.
이게 증명하는 것What this proves

전체 흐름을 한 장으로 잡고, 화면마다 기능을 번호로 정의해 개발이 추측 없이 구현하게 만드는 설계력.Capturing the whole flow on one page, then defining each screen's behavior by number so engineering builds it without guessing.

Genesis · Configurator 화면기획Genesis · Configurator spec

Concentrix Catalyst · 서비스 기획Service Design 이탈리아 마켓Italy market 2025 화면기획서(스토리보드)Storyboard

차량을 직접 구성(Configurator)하는 여정이에요. 모델 라인부터 엔진·색상·휠·인테리어·옵션을 거쳐 Summary까지 가는 단계 흐름을 먼저 정의하고, 핵심인 Summary 화면을 펼쳤습니다. 가격(부가세·등록비 포함)·금융 계산·법적 디스클레이머가 한 화면에 얽히는 곳이라, 무엇을 어떤 순서로 보여주고 어떤 문구를 띄울지를 번호 단위로 적었어요.The journey of configuring a car yourself: I defined the step flow from model line through engine, color, wheels, interior and options to the Summary, then unfolded the Summary itself. It's where price (incl. VAT and registration), finance calculation and legal disclaimers all tangle on one screen, so I specced what shows in what order and which copy appears, by number.

✎ 실제 산출물에서 발췌 · 요구사항 ID 등 기밀은 Confidential로 가렸어요.Excerpted from the real deliverable · requirement IDs and the like are masked as Confidential.
이게 증명하는 것What this proves

가격·금융·법무가 얽힌 복잡한 화면도, 보여줄 순서와 문구를 정해 하나의 명세로 끝내는 정리력.Even a screen where price, finance and legal collide, resolved into one spec by deciding the order and the copy.

Genesis · 정보구조(IA)Genesis · Information Architecture

Concentrix Catalyst · 서비스 기획Service Design 이탈리아 마켓Italy market 2025 메뉴 트리 · 오픈 단계Menu tree · go-live phasing

이탈리아 사이트(genesis.com/it)의 메뉴 구조 전체를 정의했어요. Models·Offers·Buy·Owners·Service부터 Footer까지 1~4 depth로 펼치고, 단순히 '무엇이 있나'에서 멈추지 않고 무엇을 언제 여는지까지 단계(Phase 1·2·3)로 색을 입혀 한 장에 담았습니다. 화면을 그리기 전에 이 뼈대를 먼저 합의해야, 뒤따르는 모든 화면기획이 같은 구조 위에서 움직이거든요.I defined the full menu structure of the Italy site (genesis.com/it). From Models, Offers, Buy, Owners and Service down to the footer, laid out across 1–4 depths — and not stopping at "what exists" but coloring in what opens when, by go-live phase (1·2·3), all on one page. This skeleton has to be agreed before any screen is drawn, so every spec that follows moves on the same structure.

✎ 실제 IA 산출물 · 문서 푸터·내부 식별자는 가렸어요. 색은 오픈 단계(Phase 1·2·3)를 뜻합니다.The real IA deliverable · document footer and internal identifiers masked. Colors denote go-live phase (1·2·3).
이게 증명하는 것What this proves

화면을 그리기 전에 사이트 전체 구조를 잡고, 오픈 순서까지 계획해 뒤따르는 모든 기획의 토대를 만드는 일.Setting the whole site's structure before any screen, and planning the release order, to lay the foundation every later spec stands on.
